Streamlit

Getting Started#

What's Streamlit?#

Streamlit is an open-source Python library that makes it easy to create and share beautiful, custom web apps for machine learning and data science. source_streamlit.

You may find many open-source apps written in streamlit such as e.g. the following NLP apps: a demo of an spacy NLP pipeline or an analysis of the game of thrones characters.

Prerequisites#

To follow along with this lecture you need

  • streamlit
  • pandas
  • geopandas
  • regex
  • folium
  • google-transliteration-api

Or just download the requirements.txt from the corresponding project and execute

pip install -r requirements.txt

You can install some extra transliteration libraries if you want to implement some more scripts:

  • aksharamukha
  • transliterate

Step-by-step guide to our transliteration app#

In this lecture, we will built an app that shows transliterations from latin to other scripts on a world map (showing the corresponding transliteration of the name at the place where the script is used). The choice of scripts is based on the availability of the transliteration.

How to start streamlit#

Having installed streamlit, you need to write the corresponding python programme (which you can do in your favorite IDE or text editor). We create the file "translitName.py" and start with importing streamlit and adding a title to the app:

import streamlit as st
st.title('Script your name')

Now we can invoke it with :

streamlit run translitName.py

As your command line will tell you, you can now open your streamlit app in your browser at ``http://localhost:8502´´.

How to load text in streamlit#

Now we want to read in the user input:

user_input = st.text_input("Put your name here in latin script")

How to transliterate using the google-transliteration-api#

Regex#

#The google api version that we used couldn't handle the transliteration to japanese in many cases, but by helping with onsecutive consonants (which does not exist in the japanese script, we could circumvent this issue
def japanese_rewrite(text):
if text.endswith(('t','d')):
text = text + 'o'
if text.endswith(('b','c','f','g','h','j','k','l','m','p','q','r','s','v','w','x','z')):
text = text + 'u'
japanese_rewrite0 = re.sub( r'[c][h]', r'k', text)
japanese_rewrite1 = re.sub( r"(([t|d])([bcdfghjklmnpqrstvwxz]))", r'\2o\3', japanese_rewrite0)
japanese_rewrite2 = re.sub( r"(([b|c|f|g|h|j|k|l|m|n|p|q|r|s|v|w|x|z])([b|c|d|f|g|h|j|k|l|m|n|p|q|r|s|t|v|w|x|z]))", r'\2u\3', japanese_rewrite1)
if japanese_rewrite2.__eq__(text):
return text
else:
return japanese_rewrite(japanese_rewrite2)

How to display a map using folium#

folium is a python library that can be used with streamlit. It makes it easy to visualize data that’s been manipulated in Python on an interactive leaflet map. It enables both the binding of data to a map for choropleth visualizations as well as passing rich vector/raster/HTML visualizations as markers on the map. folium source
